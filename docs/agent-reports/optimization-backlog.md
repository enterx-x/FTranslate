# FTranslate 优化点 Backlog

生成时间：2026-06-13  
执行方式：原计划由优化点子智能体生成；子智能体因工作区额度耗尽失败，当前由统筹智能体继续整理。  
原则：优先不破坏业务逻辑、不修改已有 localStorage 结构、不覆盖用户自定义表格样式。

## P0：直接影响可用性的优化

1. GitHub 推送链路不稳定：`git push` 仍可能在 HTTPS 传输层被重置；建议保留 `gh api` 备用推送方案或配置 SSH key。
2. Argos CLI 路径依赖系统 PATH：已新增 `FTRANSLATE_ARGOS_CLI` 兼容；后续设置页应展示当前 CLI 检测状态。
3. arXiv 搜索限流时用户反馈不足：需要显示冷却剩余时间、缓存命中状态和下一次可请求时间。
4. arXiv 搜索结果加载多页体验不足：需要“加载下一页”和“后台继续补齐缓存”两个清晰入口。
5. arXiv 中文检索体验不足：中文关键词应本地翻译/扩展为英文 query 后再搜 arXiv。
6. arXiv 标题/摘要翻译队列状态不够明显：应显示待翻译、翻译中、已缓存、失败可重试。
7. arXiv 结果卡片在部分窗口仍可能过密：需要按窗口宽度自动切换紧凑/列表。
8. PDFMathTranslate 错误信息仍可能直接透出原始英文/JSON：需要结构化解析错误并给出可执行建议。
9. PPT 质量检查失败时不应完全阻止导出：应允许“带质量警告导出”，同时提示问题。
10. 组会 PPT 图表精准裁剪仍未完全落地：需要把真实 PDF 图表候选插入 PPT，而不是只放占位。

## P1：高价值体验优化

11. 首页信息密度仍需按 1366x768 优化，避免卡片被截断。
12. 设置页左侧分类可读性仍可提升，长标题应显示 tooltip。
13. AI 助手拖拽栏需要边界保护，避免拖到不可恢复宽度。
14. PDF 阅读页右侧面板应记住折叠状态。
15. PDF 阅读页缩放应增加“适合宽度”和“适合页面”按钮。
16. 双语 PDF 左右模式应支持同步滚动开关。
17. 阅读笔记需要明确“保存到当前论文”和“复制 Markdown”按钮。
18. 笔记预览应默认 Markdown 渲染，不显示原始 `##`。
19. 研究表格行详情抽屉应支持复制整行 Markdown。
20. 研究表格导入 Excel 后应保留自动换行、冻结行列和基本格式。
21. 研究表格右键菜单应加入 AI 填充选区入口。
22. 研究表格格式工具栏需要更紧凑的分组和 tooltip。
23. 研究表格公式帮助应保持隐藏式，不常驻占空间。
24. 知识图谱默认节点数应按屏幕大小自适应。
25. 知识图谱右侧详情应隐藏 0 值和无意义统计。
26. 知识图谱导出图片应允许选择透明背景/白底。
27. AI 大观分析需要长任务进度条、可取消和历史恢复。
28. AI 设置应区分 OpenAI、Kimi、DeepSeek 各自支持的高级参数。
29. Kimi 非思考模型 temperature 默认值应在 UI 中明确锁定或提示。
30. API Key 保存状态应只显示“已保存”，避免暴露任何片段。

## P2：中期功能完善

31. arXiv 检索结果应支持批量选择加入备选论文库。
32. arXiv 备选论文库应有独立页面或筛选视图。
33. arXiv 搜索应支持年份范围、多分类、标题/摘要权重选择。
34. arXiv 摘要翻译可用 Argos 后台队列批量处理。
35. Argos 模型检测应支持设置页一键测试。
36. Argos 模型缺失时应能展示安装目录和模型目录。
37. 离线翻译缓存应显示 SQLite 文件位置和条目数量。
38. PPT 生成应区分“规则版”和“AI 增强版”结果。
39. PPT 生成应加入真实图表选择面板。
40. PPT 导出 Markdown 应包含每页来源页码和图表 caption。
41. PPTX 导出应支持质量检查警告列表。
42. PDF 图表候选应缓存截图，避免每次重复渲染。
43. PDF 翻译参考文献策略应在设置页和阅读页都可见。
44. 论文库详情抽屉应显示备选论文转正式入库状态。
45. 论文库应支持按笔记数量、是否有双语 PDF、是否有 PPT 筛选。
46. 设置页导出路径应支持打开目录和路径有效性检查。
47. 设置页应支持导出/导入配置 JSON。
48. UI 主题应支持浅色/深色/跟随系统，但默认保持当前黑白灰。
49. 状态栏应统一显示任务队列、缓存、API 余额、错误数量。
50. 视觉检查脚本应覆盖 arXiv 页面紧凑/标准/宽屏三种模式。

## P3：低风险持续打磨

51. 统一所有空状态插画尺寸，避免贴纸感。
52. 统一所有 card header 的标题、说明、操作按钮布局。
53. 所有危险操作增加二次确认。
54. 所有复制按钮增加成功 toast。
55. 所有长任务都写入 `docs/agent-reports` 或 app 日志目录。
56. README 增加常见故障排除：GitHub 推送、Argos、PDFMathTranslate、arXiv 429。
57. 视觉检查输出应自动生成 index.html，方便快速查看截图。
58. 打包后第一次启动应展示“环境检测”摘要。
59. 开发模式和安装版应显示当前版本号和构建时间。
60. 代码检测应增加未使用 localStorage key 和未使用 CSS class 扫描。
